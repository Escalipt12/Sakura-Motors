## 1. Общие сведения

### 1.1 Наименование системы

**Полное наименование:** Информационная система «PartStock» — маркетплейс для продажи автозапчастей.

**Краткое наименование:** PartStock.

### 1.2 Основание для разработки

Техническое задание разработано в соответствии с требованиями лабораторной работы №2 по дисциплине «Технологии программирования».

### 1.3 Цели создания системы

Проектирование и разработка маркетплейса для продажи автозапчастей с подбором по VIN-коду и параметрам автомобиля, интеграцией платёжных систем и служб доставки.

### 1.4 Описание предметной области

PartStock — маркетплейс для владельцев автомобилей, которым необходимо точное попадание запчастей в марку, модель и модификацию автомобиля. Сервис предоставляет доступ к каталогу проверенных продавцов и редких позиций, которые сложно найти в обычных гипермаркетах.

**Целевая аудитория:** владельцы автомобилей, ценящие надёжность и скорость. Им важны совместимость с конкретным авто, качество оригинальных и сертифицированных аналогов, наличие на складе и адекватная цена.

**Планируемые внешние интеграции:**
- платёжная система ЮKassa;
- сервис доставки СДЭК.

## 2. Хранимые данные

### 2.1 Сущности и атрибуты

| Сущность      | Атрибуты                                                                 |
|---------------|---------------------------------------------------------------------------|
| Товар         | Название, описание, изображения, артикул, OEM-номер, совместимость с марками/моделями авто |
| Категория     | Название                                                                  |
| Цена          | Стоимость                                                                 |
| Пользователь  | Имя, электронная почта, пароль (хэш)                                     |
| Заказ         | Адрес доставки, дата создания, сумма, статус                             |
| Статус заказа | Название                                                                  |
| Производитель | Название, страна                                                          |

### 2.2 Примеры данных

| Сущность      | Пример значения                                                          |
|---------------|---------------------------------------------------------------------------|
| Товар         | Тормозные колодки Brembo передние (VW Golf 7, Tiguan)                    |
| Категория     | Тормозная система                                                        |
| Цена          | 4 200 руб.                                                               |
| Пользователь  | Ivan, ivanov95@mumail.com                                                |
| Заказ         | Москва, 12.05.2024                                                       |
| Статус заказа | Выполнен                                                                 |

### 2.3 Связи между сущностями

- Товар принадлежит категории и может иметь несколько цен (в зависимости от продавца).
- Заказ связывает пользователя с набором товаров и имеет статус.
- Товар связан с совместимыми марками, моделями и годами выпуска автомобилей.

## 4. Функциональные требования

### 4.1 Требования к каталогу товаров

- Размещение товаров с детальными техническими характеристиками (артикул, OEM-номер, совместимость с марками/моделями авто, фото).
- Иерархическая структура категорий.
- Поиск и фильтрация по параметрам.

### 4.2 Подбор запчастей

- Подбор запчастей по VIN-коду автомобиля.
- Подбор по марке, модели и году выпуска.

### 4.3 Корзина и заказы

- Корзина покупок с учётом количества и цены.
- Оформление заказа с указанием адреса доставки.
- Отображение истории заказов.

### 4.4 Платёжные операции

- Интеграция с платёжной системой ЮKassa.
- Обработка статусов оплаты (успех, отказ, ожидание).

### 4.5 Уведомления

- Система уведомлений о статусе заказа (создан, оплачен, отправлен, доставлен).
- Уведомления по электронной почте и в личном кабинете.

## 5. Нефункциональные требования

### 5.1 Требования к интерфейсу

- Минималистичный дизайн с акцентом на визуальную составляющую товаров (фото, характеристики).
- Адаптивная вёрстка для мобильных устройств.
- Интуитивно понятная навигация.

### 5.2 Требования к безопасности

- Безопасное хранение пользовательских данных (хэширование паролей).
- Защита платёжной информации (передача данных через HTTPS, без хранения данных карт).
- Защита от основных веб-уязвимостей (XSS, CSRF, SQL-инъекции).
- Защита от мошеннических действий (валидация заказов, лимиты).

### 5.3 Требования к производительности

- Время загрузки страниц — не более 3 секунд при стандартном подключении.
- Масштабируемость архитектуры для роста нагрузки и числа пользователей.

### 5.4 Требования к надёжности

- Резервное копирование данных.
- Восстановление после сбоев в течение разумного времени.

## 6. Результаты проектирования

### 6.1 Технологический стек

| Уровень           | Технология             | Назначение                                      |
|-------------------|------------------------|-------------------------------------------------|
| Frontend          | React / Vue.js         | Пользовательский интерфейс                      |
| Backend           | Node.js (Express) / Python (Django) | Серверная логика, API                 |
| База данных       | PostgreSQL             | Хранение данных, транзакции                     |
| Кэш               | Redis                  | Сессии, кэш запросов                            |
| Хранилище файлов  | S3-совместимое / локальное | Изображения товаров                        |
| Интеграции        | REST API               | ЮKassa, СДЭК                                   |

### 6.2 Архитектура системы

```
┌─────────────────┐     HTTPS      ┌─────────────────┐
│   Браузер       │◄──────────────►│   Web-сервер    │
│   (клиент)      │                │   (Nginx)       │
└─────────────────┘                └────────┬────────┘
                                            │
                                     ┌──────▼──────┐
                                     │  Backend    │
                                     │  (API)      │
                                     └──────┬──────┘
                                            │
              ┌─────────────────────────────┼─────────────────────────────┐
              │                             │                             │
       ┌──────▼──────┐              ┌──────▼──────┐              ┌──────▼──────┐
       │ PostgreSQL  │              │   Redis     │              │  ЮKassa /   │
       │  (данные)   │              │  (кэш)      │              │    СДЭК     │
       └─────────────┘              └─────────────┘              └─────────────┘
```

**Описание:** классическая трёхуровневая архитектура (клиент — сервер приложений — база данных) с внешними интеграциями для оплаты и доставки.

### 6.3 Интерфейс пользователя

**Основные экраны:**
- Главная страница — каталог категорий, блок подбора по VIN, рекомендуемые товары.
- Каталог — список товаров с фильтрами (категория, марка, модель, цена).
- Карточка товара — фото, описание, технические характеристики, совместимость, кнопка «В корзину».
- Корзина — список выбранных товаров, итоговая сумма, оформление заказа.
- Личный кабинет — профиль, история заказов, избранное.
- Подбор по VIN — форма ввода VIN, отображение подходящих запчастей.

**Принципы проектирования интерфейса:**
- Чёткая визуальная иерархия.
- Минимум отвлекающих элементов.
- Крупные изображения товаров.
- Быстрый доступ к подбору по VIN с главной страницы.
